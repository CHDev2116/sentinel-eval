# SentinelEval — 商業說明（接案 / 提案用）

> 給決策者、採購、法遵、資安主管。**不需要懂 AI 或程式。**
>
> **English:** [Business README (EN)](BUSINESS_README.md) · 技術文件：[README.md](../README.md)

---

## 一句話

**在 AI 正式處理公司郵件之前，用模擬惡意郵件測試它會不會被騙、把危險信判成安全。**

這是 **上線前安全品管**，不是聊天機器人滿意度。

---

## 使用情境（Use Cases）

| 使用情境 | 商業價值 |
|----------|----------|
| **內部 AI 郵件助理安全測試** | 員工依賴前，先確認助理會不會被信件內容操縱 |
| **偵測隱藏指令** | 抓出藏在簽名、轉寄、頁尾、多語混寫裡的覆寫攻擊 |
| **防止提示詞覆寫（prompt override）** | 阻擋「忽略先前政策」類內容進入自動化 |
| **部署前 benchmark 本地模型** | 用**貴司郵件型態**比較候選模型，不看廠商宣傳 |
| **企業本地 LLM 評測** | 考題與跑分留在內網，降低資料外流 |
| **比較 Ollama / vLLM / LM Studio** | 同一套流程，只換模型 |
| **驗證結構化輸出** | 每封信都產出可接 SOAR、工單的 JSON |
| **AI 系統 CI 出貨門檻** | 候選版漏抓攻擊 → **建置失敗、不得上線** |
| **AI 治理流程** | 可重複證據給風險委員會與稽核 |
| **提示詞上線前評測** | 證明改 prompt 沒有讓攻擊攔截率退步 |
| **模型升級迴歸測試** | 「變聰明」卻漏更多攻擊 → 在 CI 就被攔下 |

---

## 為貴司設計 — 不是固定考卷

SentinelEval 是 **可擴充的評測框架**。開源 golden 考題是起點；接案通常會 **依貴司環境客製**：

| 您提供 | 我們配置 |
|--------|----------|
| **自訂攻擊題庫** | 貴司 thread 格式、語系、事業線釣魚樣本 |
| **企業提示詞** | 正式上線的審查 / 分類模板接入 harness |
| **內部 CI** | GitHub Actions、GitLab、Jenkins — 漏抓即 fail build |
| **本地或核准的 LLM** | 筆電 Ollama、VPC 內 vLLM、或合規 API |
| **上線標準** | 佇列、SOAR、自動結案什麼算「不可接受」 |

**給採購的訊息：** 這是 **可持續擁有的平台與方法論**，不是別人實驗室的一個分數。

## 本專案展現的能力（採購 / 能力標對照）

| 能力 | 商業意義 |
|------|----------|
| **AI 評測框架設計** | 可重複 harness，非顧問單次腳本 |
| **提示注入 / 指令覆寫測試** | 對應真實郵件威脅，非問答刷分 |
| **本地 LLM 編排** | 符合資料落地、air-gap |
| **AI CI/CD 治理** | 出貨門檻、歷史紀錄、每次改版迴歸 |
| **結構化審查管線** | JSON 接 SOAR、工單、分析 |
| **對抗表面壓力測** | 同一攻擊、多種包裝（unicode、轉寄、頁尾…） |
| **校準與信心檢視** | 「很有信心但判錯」可呈報管理層 |

## 接案交付物（合約可調）

| 交付 | 內容 |
|------|------|
| **風險對照報告** | 漏抓類型、誤判正常信 — 用佇列 / SOC / 自動化語言 |
| **上線建議** | 建議上線 · 暫緩 · 須人工覆核 |
| **改版前後比較** | 換模型 / 改 prompt 的證據 |
| **客製題庫** | 對齊貴司郵件流程（可選） |
| **CI 導入說明** | 嵌入 release 流程（可選） |
| **主管 + 工程簡報** | 決策摘要 + 結構化 JSON 報告 |

## 客戶常見情境

| 情境 | 未先測的風險 |
|------|--------------|
| AI 自動分類 / 結案郵件 | 隱藏指令 → **錯誤佇列、該擋未擋** |
| AI 幫 SOC 讀信 | 摘要好看但結論錯 → **通報延誤** |
| 換新模型「變聰明」** | 攻擊變鈍 → **上線後才爆** |
| 雲端代測 | 樣本外流 → **合規風險** |

---

## 避免的損失

1. 漏抓攻擊仍上線  
2. 錯誤自動化路由或結案  
3. 升級後沉默退步  
4. 過度自信卻判錯  

---

## 與一般 AI「準確度」評測的差異

| 一般評測 | SentinelEval |
|----------|--------------|
| 問答對錯 | **郵件藏指令仍判安全？** |
| 上傳雲端 | **貴司環境執行** |
| 一個總分 | **攻擊攔截 · 誤殺 · 逐案出貨門檻** |
| 一次性 | **可重複、可稽核** |

## 隱私與合規

- 預設 **合成考題**  
- **境內 / 本機執行**  
- **防禦性驗證** — 模擬攻擊用於測試  

（DPA 與控制措施依合約。）

## 建議合作流程

1. 工作坊 — 定流程與不可接受錯誤  
2. 約定上線標準  
3. 跑評測 + 解讀  
4. 迭代至達標或列殘餘風險與補償控制  

---

## 名詞對照

| 詞 | 白話 |
|----|------|
| Prompt injection | 信裡藏「叫 AI 不要守規則」 |
| Instruction override | 「忽略先前指示…」 |
| Release gate | 測試未過不得上正式流量 |
| Golden 考題 | 每次改版用同一標準答案比較 |

---

## 適合 · 不適合

**適合：** 郵件/工單/SOC 導入 AI；要向管理層證明治理；合約要寫驗收。  

**不適合：** 只要聊天滿意度；內容不驅動自動化；只要分數不定義失敗。
